# 수정지시서 002 실행결과

## 항목별 처리

| # | 지시 항목 | 처리 | 비고 |
|---|---|---|---|
| 1 | Target 인터페이스 정의 | `target.go`에 `Target` 인터페이스 + `DefaultTarget()` 생성 | `GeneratePage`, `FileExtension`, `Dependencies` 3개 메서드 |
| 2 | React 코드 이동 | `react_target.go` 생성, `imports.go` → `react_imports.go`, `templates.go` → `react_templates.go` 이름 변경 | React 전용 코드 격리 완료 |
| 3 | generator.go 리팩토링 | 위임 래퍼 + `GenerateWith()` + 공통 유틸만 유지 | 기존 API 시그니처 불변 |
| 4 | ReactTarget.Dependencies | `collectImports` 기반 동적 의존성 수집 | react-query, react-hook-form, react-router-dom |
| 5 | GenerateOptions.Target 필드 | 미채택 | `GenerateWith(target, ...)` 방식으로 충분. 옵션에 Target 필드를 넣으면 책임이 중복됨 |

## 변경 파일

| 파일 | 유형 | 설명 |
|---|---|---|
| `generator/target.go` | NEW | Target 인터페이스, DefaultTarget(), 컴파일 검증 |
| `generator/react_target.go` | NEW | ReactTarget struct + GeneratePage/FileExtension/Dependencies + renderFetchHooks |
| `generator/react_imports.go` | RENAME | imports.go에서 이름 변경. 내용 변경 없음 |
| `generator/react_templates.go` | RENAME | templates.go에서 이름 변경. 내용 변경 없음 |
| `generator/generator.go` | 수정 | Generate/GeneratePage → DefaultTarget 위임, GenerateWith 추가, mergeOpt 추가 |
| `generator/generator_test.go` | 수정 | TestGenerateWithDefaultTarget 추가 (+1) |

## 지시서 대비 차이점

| 지시 내용 | 실제 처리 | 이유 |
|---|---|---|
| `GenerateOptions.Target` 필드 추가 | 미채택 | `GenerateWith()` 함수로 Target 지정이 가능하므로 옵션에 중복 필드 불필요 |
| `renderFetchHooks` generator.go에 유지 | `react_target.go`로 이동 | React useState/useQuery 생성 함수이므로 React Target에 속하는 것이 자연스러움 |
| `outName` 생성에 `FileName` 사용 | `page.Name` 사용 | PageSpec에 FileName 필드 없음. 기존 코드와 동일하게 Name 사용 |

## fullend 호출 예시

```go
// 기존 방식 (변경 없이 동작)
result, err := generator.Generate(pages, specsDir, outDir, generator.GenerateOptions{
    APIImportPath: "../api",
    UseClient:     false,
})

// 새 방식 (Target 명시)
result, err := generator.GenerateWith(generator.DefaultTarget(), pages, specsDir, outDir)
```
